# Release Notes

This file contains update messages for each version. Users see these messages when updating.

## Version Format

Each version entry should include:
- Version number (matching package.json)
- Update message to display to users
- Date of release

## Release Updates

### Future Versions

Add new versions above this line with format:
```markdown
### X.Y.Z - YYYY-MM-DD
Update: Your update message here.

Additional details if needed...
```

## Usage Guidelines

- Add new version entries when preparing release
- Match version number to package.json
- Include date in YYYY-MM-DD format
- Keep messages concise and user-friendly
- For VSCode extensions: Focus on features and bug fixes that impact users
- Reference CHANGELOG.md for detailed technical changes

## Example Entry

```markdown
### 1.0.2 - 2026-02-01
Update: Enhanced configuration panel with new options.

This version includes:
- Added auto-refresh toggle with customizable interval
- Improved error handling for API authentication
- Fixed tooltip display for high-usage scenarios
- Enhanced status bar icons with new warning states
```
